# 2310-轮位压缩筛 (Wheel-2310 Bitwise Sieve)

**目录：** `algorithms/wheel-2310-sieve/`

## 算法简介

使用前五个质数 **2、3、5、7、11** 构造模 **2310**（= 2×3×5×7×11）的质数轮，
并结合位压缩技术实现高效筛法。

### 核心思想

1. **质数轮剪枝**：φ(2310) = 480，每 2310 个连续整数中仅有 **480 个候选数**（约 20.8%），
   相比 210-轮的 22.9% 再减少约 9%，直接降低外层筛选的工作量。

2. **位压缩存储**：将每个 2310-数块的 480 个候选标志压缩存储在 **8 个 64 位整数**（512 位中的 480 位）中。
   对于 n = 1,000,000，整个筛数组约 **27 KB**，可完全装入 L1 缓存（典型大小 32–64 KB），
   极大减少内存带宽压力。

3. **增量模运算**（无大型乘积表）：内层标记循环通过增量方式跟踪 `composite % 2310`，
   每步仅需一次加法与一次条件减法，配合 2310 项余数索引表（4.6 KB，可驻留 L1 缓存）；
   为每个待筛素数预计算 480 项步长数组（1.9 KB），避免构造 480×480 ≈ 900 KB 的静态乘积表。

### 与 210-轮筛的对比

| 指标 | 210-轮 | 2310-轮 |
|------|--------|---------|
| 轮模数 | 210 | 2310 |
| 候选比例 | 48/210 ≈ 22.9% | 480/2310 ≈ 20.8% |
| 每块 ulong 数 | 1 | 8 |
| n=1M 筛数组大小 | ~38 KB | ~27 KB |
| 乘积表大小 | 48×48×4 = 9 KB | 无（增量计算） |

## 复杂度

- **时间复杂度**：O(n log log n)
- **空间复杂度**：O(n / 2310 × 480 位) ≈ O(n / 38.5)，相当于每位表示约 4.8 个整数
- **适用场景**：批量生成范围内所有质数，尤其适合 n ≤ 10⁷ 的场景（筛数组可完全驻留缓存）
